Project overview

The project connects the SensingSP ray tracer to a research-grade WiFi sensing workflow. wifisensing_generateCSI.py converts propagation paths into complex CSI and stores the tensor as Total_H_Signal.npy. The processor then works directly on that simulated CSI.

Signal model

The simulated channel combines static and target-dependent propagation, with optional timing and carrier-frequency offsets:

\[ \widetilde{H}_{i,j,k}=e^{\mathrm{j}\phi^{\mathrm{offset}}_{j,k}} \left(H^{\mathrm S}_{i,j}+H^{\mathrm D}_{i,j,k}\right)+N_{i,j,k}. \]

For a dominant moving path, the dynamic component is modeled across antenna, subcarrier, and slow time as

\[ H^{\mathrm D}_{i,j,k}=A_{i,j,k} \exp\!\left[-\mathrm{j}\frac{2\pi f_j}{c} \left(d_0+(i-1)\Delta d\sin\theta+\dot d\,k\Delta t\right)\right]. \]

Multi-point differencing removes the static component over a short window:

\[ X(i,j,k)=H^{c}_{i,j,k}-H^{c}_{i,j,k_0} \approx \lambda\,a_i\circ b_j\circ c_k. \]
Implementation detail. The CP time factor has arbitrary complex phase. The processor therefore fits a general complex circle before unwrapping its phase, making Doppler estimation invariant to CP phase rotation.

Processing pipeline

1. Phase reference

Optionally extract the dominant LoS direction with MVDR and compensate common packet phase.

2. Static suppression

Subtract the first CSI sample in each window from all samples to form MD-CSI.

3. Tensor separation

Apply complex rank-one CP-ALS to obtain spatial, frequency, and temporal factors.

4. Doppler

Fit the time factor to a complex circle, unwrap the centered phase, and estimate its slope.

5. AoA

Match the spatial factor to a ULA steering dictionary using one-atom OMP.

6. Path parameter

Match the frequency factor to delay hypotheses and recover the dominant excess path.

Reproduce the experiment

First generate the complex CSI tensor inside Blender with SensingSP:

# Run inside Blender's Python environment
exec(open("wifisensing_generateCSI.py").read())

# Output shape: (3 RX, 30 subcarriers, time samples)
# File: Total_H_Signal.npy

Then run the standalone MD-CSI processor:

python wifi_md_csi_processing.py Total_H_Signal.npy \
  --fs 1000 \
  --fc 5.32e9 \
  --window 128 \
  --hop 64 \
  --compensation none \
  --output-prefix md_csi_results

The command writes a CSV containing one estimate per window and a PNG summary figure.

  • Use --compensation none for the current noiseless simulation.
  • Enable MVDR compensation after adding time-varying CFO/TO packet phase.
  • Add complex AWGN after summing all propagation paths.
  • Keep the same CFO/TO realization across RX antennas for each packet.

Recommended next experiment
